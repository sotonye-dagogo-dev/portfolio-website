# PortfolioWebsite

**The portfolio website for _Sotonye Dagogo_, software developer.**

Live [here](https://sotonye-dagogo.is-a.dev/)

## Framework

- **Angular 19.2.x** (upgraded from 18.0.0) — standalone components, SSR
- **Tailwind CSS v3.4** + **SCSS** — styling
- **FontAwesome 1.x** — icons

## Content Architecture

All page content is driven by `src/app/content/content.service.ts`, which loads static TypeScript configs as compile-time defaults and optionally merges runtime enrichment from `public/content/content.generated.json` (generated by `scripts/enrichment.js` from GitHub API data).

| Config File              | Purpose                                             |
| ------------------------ | --------------------------------------------------- |
| `site.config.ts`         | Site-wide copy, nav links, social links, stats      |
| `about.config.ts`        | About page (bio, how-I-build, contact, currently)   |
| `experience.config.ts`   | Timeline entries (HICC, Liquiditti, Along, etc.)    |
| `projects.config.ts`     | Four-tier project list (flagship, applied, archive) |
| `automation.config.ts`   | Flow diagram steps + feature cards                  |
| `tech-stack.config.ts`   | Technology badges                                   |
| `certificates.config.ts` | Certificate categories and items                    |

## AI-Assisted Development System (`ai-system`)

This repository includes the `ai-system` framework for AI-assisted software development — a vendor-neutral, model-agnostic system providing structured documentation, command-driven workflows, and quality gates.

**Key directories:**

- `ai-system/protocols/` — Entry, tiering, QA, escalation, verification protocols
- `ai-system/agents/` — Function-based roles (Planner, Architect, Implementer, etc.)
- `ai-system/commands/` — Reusable command pipelines (execute-feature, dev-cycle, etc.)
- `ai-system/standards/` — Engineering principles
- `ai-system/planning/` — Task queue and project plan
- `ai-system/memory/` — Decisions and lessons
- `ai-system/index/` — Repo map and dependency graph
- `ai-system/testing/` — Test plan and results
- `ai-system/checkpoints/` — Session log and in-progress tracking
- `ai-system/summaries/` — Development history

Start a session by reading `.ai-context.md`.

## Scripts

- `scripts/enrichment.js` — Reads repo slugs from `projects.config.ts`, calls GitHub API, writes `public/content/content.generated.json`
- `npm run dev` — Development server
- `npm run build` — Production build (SSR + prerender)
